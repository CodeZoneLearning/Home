# APIs como contrato

Esta aula move o conteúdo de APIs para dentro do subject de Python, com foco em conceito antes de framework.

## O que estudar

- API como contrato entre sistemas.
- Requisição e resposta HTTP.
- Métodos, recursos, headers, body e status codes.
- Schemas, validação, mensagens de erro e versionamento.
- Python como cliente e como servidor.

## Prática recomendada

Antes de implementar uma API, escreva exemplos de:

- request válido;
- response de sucesso;
- response de erro de validação;
- response para recurso inexistente;
- campos obrigatórios e opcionais;
- regras de compatibilidade entre versões.
