01MODELO MENTAL

// EXPERIMENTO CONTÉM RUNS

Uma métrica isolada não identifica um treinamento.

Para reproduzir um resultado, precisamos saber qual código, dataset, ambiente, divisão, parâmetros e artefatos produziram aquele número.

EXPERIMENTequipment-health

Agrupa runs que respondem à mesma pergunta de modelagem.

contains
RUNrf_depth_06

Uma execução com início, fim, status e identificador único.

records
EVIDENCEcontext + results

Parâmetros, métricas, tags, datasets, artefatos e modelos.

INPUT CONTEXT

O que entrou

Versão dos dados, features, split, seed, dependências e código.

TRAINING CHOICES

O que foi decidido

Algoritmo, hiperparâmetros, pré-processamento e threshold.

OBSERVED RESULTS

O que aconteceu

Métricas, curvas, matriz de confusão, erros e modelo produzido.

RUN NÃO É MODELO

Uma run registra uma execução inteira. O modelo é apenas um dos possíveis artefatos produzidos por ela.

02REGISTRAR

// TIPO DE DADO DEFINE A FUNÇÃO

Registre intenção, resultado e evidência.

A interface de tracking separa valores por função. Essa disciplina torna filtros e comparações mais confiáveis.

PARAMS

`log_param`

max_depth = 6

Configurações tratadas como entrada da run. Normalmente não mudam durante a execução.

METRICS

`log_metric`

recall = 0.90

Resultados numéricos que podem ser registrados por etapa ao longo do tempo.

ARTIFACTS

`log_artifact`

confusion_matrix.png

Arquivos: relatórios, gráficos, previsões, exemplos e diagnósticos.

TAGS

`set_tag`

purpose = baseline

Metadados pesquisáveis sobre intenção, proprietário, commit ou estado.

CALCULAR

scikit-learn

recall = recall_score(y_test, predictions)

Define a fórmula e calcula o valor usando targets e previsões.

REGISTRAR

MLflow Tracking

mlflow.log_metric("recall", recall)

Persiste o valor junto ao contexto da run. Não recalcula nem valida a métrica.

04REGISTRY

// IDENTIDADE E PROMOÇÃO

Registrar cria uma versão; promover exige decisão.

O Model Registry organiza modelos registrados e suas versões. Aliases como `champion` e `challenger` oferecem referências estáveis sem acoplar consumidores a um número fixo.

LOGGED MODELruns:/.../model

Artefato associado a uma run e à linhagem de treinamento.

→
REGISTERED MODELequipment-health

Nome estável que reúne versões do mesmo contrato.

→
MODEL VERSIONversion 7

Versão imutável criada a partir de um modelo logado.

→
ALIAS@challenger

Referência mutável controlada por política de promoção.

equipment-health

Aliases atuais

VERSION 06champion

Modelo atualmente aprovado para produção.

VERSION 07challenger

Candidato em validação antes de receber tráfego.

REGISTRY NÃO É CI/CD

O registro mantém identidade e metadados. Testes, aprovações, deployment e rollback ainda precisam de um processo controlado.

05LABORATÓRIO

// TRACKING LOCAL COM SQLITE

Treine, registre e abra a interface.

O exemplo usa um classificador de falha, registra o dataset, parâmetros, cinco métricas, relatório JSON, assinatura, exemplo de entrada e uma nova versão no Registry.

TRACKING API / UI

Consulta e comparação

Cliente Python grava runs; a interface permite filtrar e comparar.

BACKEND STORE

`mlflow.db`

ARTIFACT STORE

`mlruns/`

Arquivos maiores, relatórios e modelos ficam fora das tabelas do backend.

train_with_mlflow.py
with mlflow.start_run(run_name="random_forest"):
    mlflow.log_params({"max_depth": 6, "n_estimators": 200})
    model.fit(X_train, y_train)
    predictions = model.predict(X_test)
    recall = recall_score(y_test, predictions)
    mlflow.log_metric("recall", recall)
    mlflow.sklearn.log_model(
        sk_model=model,
        artifact_path="model",
        registered_model_name="equipment-health-classifier",
    )
