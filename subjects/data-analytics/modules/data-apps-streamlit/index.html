streamlit_minimal.py
import pandas as pd
import streamlit as st

st.title("Learning Analytics App")

df = pd.read_csv("data/learning_activity.csv")
track = st.sidebar.multiselect(
    "Trilhas",
    sorted(df["track"].unique()),
    default=sorted(df["track"].unique()),
)

filtered = df[df["track"].isin(track)]
st.metric("Conclusão média", f"{filtered['completion_rate'].mean():.1%}")
st.dataframe(filtered)
03CACHE

// PERFORMANCE E CONFIANÇA

Cache acelera. Validação protege.

Streamlit reexecuta o script a cada interação. Isso é simples e produtivo, mas leitura de arquivos e agregações podem ficar caras. `@st.cache_data` guarda resultados enquanto a entrada não muda.

cache_and_schema.py
REQUIRED_COLUMNS = {"week", "learner_id", "track", "completion_rate"}

@st.cache_data(show_spinner=False)
def load_default_data(path: str) -> pd.DataFrame:
    return pd.read_csv(path)

def validate_schema(data: pd.DataFrame) -> None:
    missing = REQUIRED_COLUMNS.difference(data.columns)
    if missing:
        raise ValueError(f"Colunas ausentes: {sorted(missing)}")

05PUBLICAR

// PROXIMO PASSO

Publicar exige empacotar o app corretamente.

Para sair da máquina local, o projeto precisa declarar dependências, entrada da aplicação, dados de exemplo e instruções claras. O código também deve tratar erros sem expor detalhes desnecessários ao usuário final.

requirements.txt

Dependências

`streamlit`, `pandas` e `plotly` precisam estar declarados.

app.py

Ponto de entrada

Arquivo executado por `streamlit run app.py`.

data/

Exemplo reproduzível

CSV padrão permite abrir o app sem upload inicial.

README.md

Contrato de uso

Explica execução, schema esperado, limites e próximos passos.

Prática guiada

  1. Execute `streamlit run app.py`.
  2. Filtre apenas duas trilhas e observe as métricas.
  3. Aumente a pontuação mínima e interprete o que mudou.
  4. Envie outro CSV com uma coluna ausente e observe a validação.
  5. Baixe os dados filtrados e compare com a tabela da tela.
