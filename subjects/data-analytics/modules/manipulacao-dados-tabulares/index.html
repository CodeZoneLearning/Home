01CONTEXTO

// A PERGUNTA

O equipamento mudou ou o teste falhou?

Cada equipamento foi testado seis vezes. Se todas as classificações apontam para a mesma classe, existe consenso. Se cada teste produz uma resposta diferente, o processo de medição ou o classificador precisa ser investigado.

1
HealthyOperação normal
2
AttentionMonitorar sinais
3
DegradedPlanejar intervenção
4
CriticalAção imediata
Importante

`confidence_score` é a confiança fictícia do classificador. Ela não prova que a classe está correta; apenas mostra o quanto o modelo afirma confiar na própria previsão.

03GROUPBY

// UMA LINHA POR EQUIPAMENTO

Meça o grau de concordância.

A granularidade muda de um teste por linha para um equipamento por linha. A moda representa a classe dominante e a taxa de concordância mede quantos testes sustentam essa classe.

groupby.py
summary = df.groupby("equipment_id").agg(
    tests_count=("test_id", "count"),
    unique_statuses=("health_status", "nunique"),
    dominant_status=("health_status", lambda x: x.mode().min()),
    mean_confidence=("confidence_score", "mean")
)

dominant_count = df.groupby("equipment_id")["health_status"] \
    .apply(lambda x: x.value_counts().max())

summary["agreement_rate"] = dominant_count / summary["tests_count"]

04INFERÊNCIA

// INCERTEZA EXPLÍCITA

Compare médias com intervalos.

Agrupar a confiança por resultado permite investigar um padrão: testes contraditórios também apresentam menor confiança? O intervalo de 95% mostra a precisão da média estimada em cada grupo.

Carregando relatório estatístico...

IC 95% = média ± 1,96 × erro padrão

Aproximação normal didática. Uma validação industrial precisaria de amostra representativa, referência real de falha e análise temporal.

LEITURA DO CASO

Divergência e baixa confiança aparecem juntas nesta base.

Os equipamentos consistentes têm confiança média próxima de 0,92. Nos inconsistentes, ela cai para aproximadamente 0,64. Isso é um sinal para investigação, não prova de causalidade.
