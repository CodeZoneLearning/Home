01DESENHO

// ANTES DO CÓDIGO

Dashboard não é coleção de gráficos.

Um dashboard precisa de hierarquia: primeiro os indicadores que orientam a leitura, depois os gráficos que explicam diferenças, tendências e relações.

Usuário

Quem vai olhar?

Uma pessoa que acompanha evolução das trilhas e quer saber onde investigar.

Decisão

O que muda depois da leitura?

Priorizar suporte, revisar trilhas e acompanhar desempenho semanal.

Camadas

O que vem primeiro?

KPIs no topo, comparação e tendência no meio, exploração e correlação abaixo.

Interação

O que deve ser filtrável?

Trilhas podem ser isoladas por dropdown e legenda sem recriar o dashboard.

02SUBPLOTS

// FIGURA COMPOSTA

`make_subplots` organiza o painel.

Plotly permite montar uma figura com diferentes tipos de visualização. O dashboard desta aula usa quatro indicadores, barras, linhas, dispersão e heatmap em uma única figura.

KPI 01Participantes
KPI 02Conclusão
KPI 03Pontuação
KPI 04Suporte
BarrasConclusão por trilha
LinhaPontuação semanal
DispersãoHoras x desempenho
HeatmapCorrelação
dashboard_layout.py
from plotly.subplots import make_subplots
import plotly.graph_objects as go

fig = make_subplots(
    rows=3,
    cols=4,
    specs=[
        [{"type": "indicator"}, {"type": "indicator"}, {"type": "indicator"}, {"type": "indicator"}],
        [{"type": "bar", "colspan": 2}, None, {"type": "scatter", "colspan": 2}, None],
        [{"type": "scatter", "colspan": 2}, None, {"type": "heatmap", "colspan": 2}, None],
    ],
)

fig.add_trace(go.Indicator(mode="number", value=192, title={"text": "participantes"}), row=1, col=1)
03INTERAÇÃO

// HOVER, LEGENDA E FILTRO

Interatividade precisa ter propósito.

Hover revela detalhes sem poluir o gráfico. Legenda permite esconder séries. Dropdown troca o foco da leitura. Tudo isso deve reduzir esforço, não esconder a mensagem principal.

Hover

Detalhe sob demanda

Semana, tarefas, conclusão e satisfação aparecem apenas quando o usuário investiga um ponto.

Legenda

Comparar ou isolar

Clicar na legenda ajuda a reduzir ruído visual em gráficos com múltiplas trilhas.

Dropdown

Foco por trilha

O dashboard pode mostrar todas as trilhas ou destacar uma sem recalcular os dados.

plotly_interaction.py
buttons = [
    {"label": "Todas as trilhas", "method": "update", "args": [{"visible": [True] * len(fig.data)}]},
    {"label": "Python", "method": "update", "args": [{"visible": python_visibility}]},
]

fig.update_layout(
    updatemenus=[{"type": "dropdown", "buttons": buttons}],
    hovermode="closest",
)

04ENTREGA

// HTML STANDALONE

Plotly entrega um artefato interativo.

O dashboard pode ser salvo como HTML para revisão, compartilhamento e versionamento. Isso não substitui um data app, mas resolve bem relatórios exploratórios e entregas leves.

Arquivo

`learning_analytics_dashboard.html`

Dashboard completo com Plotly carregado via CDN.

Reprodução

`generate_dashboard.py`

Script único para recriar o HTML e os resumos.

Limite

HTML não é aplicação

Para upload, autenticação e estado, use Streamlit ou outra camada de app.

05PRÁTICA

// EXERCÍCIOS

Troque a pergunta, não só a cor.

A prática é reorganizar o dashboard quando o objetivo muda. Se a decisão muda, a hierarquia visual também deve mudar.

A

Foco em suporte

Coloque pedidos de suporte como KPI principal e crie uma visão por semana.

B

Foco em satisfação

Substitua o heatmap por boxplot de satisfação por trilha.

C

Foco executivo

Remova a dispersão e deixe apenas KPIs, barras e linha temporal.

D

Foco exploratório

Adicione mais informações no hover e justifique o que deve aparecer.
