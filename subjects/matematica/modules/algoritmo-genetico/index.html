01GENES

// COMO UMA SOLUÇÃO VIRA CROMOSSOMO?

O primeiro projeto é a representação.

Um algoritmo genético manipula candidatos. Para um problema de seleção de itens, uma solução pode ser um vetor binário `x`, em que `x_i=1` significa escolher o item `i` e `x_i=0` significa deixar de fora.

CROMOSSOMO BINÁRIO
x ∈ {0,1}n

Se `x = [1,0,1,1]`, o candidato escolhe os itens 1, 3 e 4. Essa codificação é simples, mas nem sempre é suficiente: rotas, ordens e agendas podem exigir cromossomos por permutação.

GENE

Uma decisão local

Escolher item, ativar parâmetro, usar recurso ou definir uma parte da solução.

CROMOSSOMO

Solução completa

Vetor, lista, permutação ou árvore que representa uma candidata possível.

POPULAÇÃO

Várias hipóteses

Conjunto de candidatos avaliados e transformados a cada geração.

CUIDADO

Uma representação ruim cria candidatos inválidos demais ou impede soluções boas de aparecer. Antes de ajustar taxa de mutação, confira se o cromossomo expressa o problema corretamente.

02FITNESS

// O QUE SIGNIFICA SER MELHOR?

Fitness transforma solução em pontuação.

Em otimização, normalmente queremos minimizar custo ou maximizar valor. Para restrições, uma prática comum é penalizar soluções inválidas em vez de descartá-las imediatamente.

MOCHILA BINÁRIA
max Σ vixi sujeito a Σ cixi ≤ B

`v_i` é valor, `c_i` é custo e `B` é orçamento. O vetor binário define quais itens entram na mochila.

FITNESS COM PENALIDADE
F(x)=Σ vixi - λ max(0, Σ cixi - B)2

A penalidade quadrática pune violações grandes com mais força. O parâmetro `λ` controla o quanto a restrição pesa contra o valor bruto.

MAXIMIZAR

Maior fitness vence

Valor, lucro, cobertura, acurácia ou utilidade podem virar pontuação positiva.

MINIMIZAR

Use sinal ou inverso

Para custo, erro ou distância, pode usar `F=-custo` ou uma transformação equivalente.

RESTRIÇÕES

Penalize violações

Orçamento, capacidade, prazo e regras de negócio precisam aparecer no fitness.

03EVOLUIR

// COMO A POPULAÇÃO MUDA?

Seleção explora bons candidatos; mutação preserva diversidade.

O ciclo típico é avaliar, selecionar pais, cruzar genes e mutar alguns pontos. Elitismo preserva os melhores candidatos para evitar perder progresso por acaso.

SELEÇÃO

Torneio

Sorteie alguns candidatos e escolha o melhor entre eles. É simples, robusto e controla pressão seletiva pelo tamanho do torneio.

CRUZAMENTO

Um ponto

Escolha um corte `k` e combine prefixo de um pai com sufixo do outro: `filho = paiA[:k] + paiB[k:]`.

MUTAÇÃO

Bernoulli por gene

Cada gene tem probabilidade `p_m` de inverter. Em binário: `0` vira `1`, `1` vira `0`.

PROBABILIDADE DE MUTAÇÃO
P(x'j ≠ xj) = pm

Mutação baixa demais pode estagnar; alta demais transforma a busca em ruído. O valor útil depende do tamanho do cromossomo e do custo de avaliar candidatos.

SEM GARANTIA GLOBAL

Algoritmo genético é metaheurística. Ele entrega uma boa solução encontrada dentro de um orçamento de gerações, não uma prova matemática de ótimo.

05CÓDIGO

// IMPLEMENTAÇÃO ESSENCIAL

O loop evolutivo é curto, mas as escolhas importam.

A maior parte da qualidade vem de representação, fitness e parâmetros. O esqueleto abaixo mostra o ciclo básico.

algoritmo_genetico.py
for geracao in range(geracoes):
    populacao.sort(key=fitness, reverse=True)
    elite = populacao[:2]
    nova = elite[:]
    while len(nova) < tamanho_populacao:
        pai_a = torneio(populacao)
        pai_b = torneio(populacao)
        filho = cruzar(pai_a, pai_b)
        nova.append(mutar(filho, taxa_mutacao))
    populacao = nova
