01SOLVER

// CONTRATO ANTES DO MÉTODO

Todo solver precisa saber o que procurar.

Antes de escolher o algoritmo, defina a representação do problema. O solver não “entende” o mundo: ele só compara estados, custos e regras de avanço.

ESTADO

Onde estou?

Nó de um grafo, configuração de uma rota, vetor de parâmetros ou solução candidata.

VIZINHOS

Para onde posso ir?

Transições permitidas, movimentos válidos, mutações ou novas soluções geradas.

CUSTO

Quanto isso custa?

Distância, tempo, risco, perda, energia, dinheiro ou penalidade combinada.

OBJETIVO

Quando paro?

Chegar ao destino, minimizar erro, bater limite de orçamento ou esgotar iterações.

IDEIA GERAL

Um solver busca `argmin custo(solucao)`. Em grafos, ele expande caminhos. Em otimização, ele melhora candidatos. O formato muda, mas a pergunta é a mesma: qual solução tem menor custo aceitável?

02DIJKSTRA

// MENOR CAMINHO COM CUSTOS POSITIVOS

Dijkstra sempre expande o menor custo acumulado.

Dijkstra é uma escolha robusta quando todas as arestas têm custo não negativo e não existe uma boa heurística. Ele mantém uma fronteira de candidatos e sempre processa o caminho com menor custo conhecido até agora.

VANTAGEM

Garante menor caminho com custos não negativos.

CUSTO

Pode explorar muito espaço quando o destino está longe.

USO

Rotas, redes, dependências, logística e planejamento sem heurística confiável.

03A*

// CUSTO ACUMULADO + ESTIMATIVA

A* usa uma heurística para mirar o destino.

A* escolhe o próximo estado pelo menor `f(n) = g(n) + h(n)`. `g(n)` é o custo real até o nó. `h(n)` é uma estimativa do custo restante. Se a heurística não superestima o custo real, A* preserva a garantia de caminho ótimo.

g(n)custo já pago

Distância acumulada da origem até o nó atual.

+
h(n)estimativa restante

Distância em linha reta, Manhattan ou outra regra admissível.

=
f(n)prioridade

Valor usado para ordenar a fronteira de busca.

04GENÉTICO

// QUANDO NÃO HÁ CAMINHO DIRETO

Algoritmo genético melhora uma população de soluções.

Em vez de expandir um grafo, o algoritmo genético mantém candidatos, mede fitness, seleciona os melhores, mistura características e aplica mutações. Ele não garante ótimo global, mas é útil em problemas grandes, discretos, ruidosos ou sem derivada confiável.

SELEÇÃO

Priorize candidatos com menor custo ou maior fitness.

CRUZAMENTO

Combine partes de duas soluções para criar novas candidatas.

MUTAÇÃO

Altere candidatos para escapar de regiões ruins do espaço.

GENÉTICO NÃO É MÁGICA

Ele depende de representação, fitness, diversidade e orçamento de iteração. Use quando métodos exatos ou gradiente forem caros demais ou inadequados.

05CÓDIGO

// ESCOLHA PELO PROBLEMA

O solver certo depende da estrutura disponível.

Se há grafo com pesos positivos, Dijkstra é baseline. Se há destino e boa estimativa de distância, A* reduz expansões. Se a solução é uma configuração difícil de navegar diretamente, algoritmo genético pode explorar alternativas.

SOLVERDijkstra

Ótimo com custos não negativos, sem precisar de heurística.

SOLVERA*

Ótimo com heurística admissível e geralmente expande menos nós.

SOLVERGenético

Busca aproximada para espaços grandes, discretos ou sem derivada.
