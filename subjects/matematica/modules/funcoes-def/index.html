01MODELO

// ENTRADA → REGRA → SAÍDA

Função separa cálculo de valor.

Uma expressão calcula uma vez. Uma função nomeia uma regra para muitos valores. Isso permite comparar cenários, testar bordas e reaproveitar a mesma transformação.

ENTRADA

O que varia?

Preço, quantidade, desconto, peso, erro, sensor ou qualquer número que muda entre cenários.

REGRA

Qual relação existe?

Multiplicar, somar taxa fixa, elevar ao quadrado, aplicar penalidade ou combinar pesos.

SAÍDA

O que quero medir?

Custo total, lucro, risco, erro médio, score ou valor previsto.

FUNÇÃO MATEMÁTICA

Uma função associa cada entrada válida a uma saída bem definida. Se a regra falha para certos valores, esse domínio precisa ser declarado.

02DEF

// UMA REGRA COM NOME

`def` registra a transformação.

O corpo da função deve fazer uma coisa clara. Entradas entram pelos parâmetros; a saída volta pelo `return`. Evite depender de variáveis soltas quando o valor pode ser argumento.

EXPRESSÃO SOLTA
subtotal = 12 * 34.90
total = subtotal + 18.00

Resolve um caso, mas esconde a regra.

FUNÇÃO
def custo_total(qtd, preco, frete):
    return qtd * preco + frete

Explica a regra e aceita muitos casos.

RETURN NÃO É PRINT

`print` mostra algo na tela. `return` entrega um valor para outro cálculo. Funções matemáticas devem retornar valores.

03PARÂMETROS

// CONTROLE EXPLÍCITO DA REGRA

Parâmetros tornam hipóteses editáveis.

Uma função mais elaborada ainda deve ser legível. Use nomes descritivos, defaults para hipóteses comuns e validações quando valores inválidos mudam o sentido matemático.

função com validação
def custo_total(quantidade, custo_unitario, custo_fixo=0):
    if quantidade < 0 or custo_unitario < 0 or custo_fixo < 0:
        raise ValueError("custos e quantidade não podem ser negativos")
    return custo_fixo + quantidade * custo_unitario

05CUSTO

// FUNÇÃO PARA COMPARAR DECISÕES

Função de custo transforma erro em número.

Otimização precisa de uma função objetivo. Em modelos, uma escolha comum é medir o erro quadrático médio: previsões muito erradas recebem penalidade maior.

ERRO ABSOLUTO|y − ŷ|

Penaliza cada unidade de erro de forma linear.

ERRO QUADRÁTICO(y − ŷ)²

Aumenta muito o custo de erros grandes.

CUSTO TOTALmédia dos erros

Resume muitos exemplos em um único número comparável.

funcoes_custo.py
def mse(observado, previsto):
    if len(observado) != len(previsto):
        raise ValueError("listas devem ter o mesmo tamanho")
    erros = [(y - y_hat) ** 2 for y, y_hat in zip(observado, previsto)]
    return sum(erros) / len(erros)
