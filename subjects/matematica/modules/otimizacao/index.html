01OBJETIVO

// O QUE ESTAMOS PROCURANDO?

Otimização começa definindo a função.

A pergunta não é “qual método usar?”, mas “qual número mede uma decisão melhor?”. Pode ser custo financeiro, erro de previsão, tempo, risco ou uma penalidade combinada.

VARIÁVEL DE DECISÃO

O que posso escolher?

Quantidade produzida, preço, limiar de alerta, parâmetro de modelo ou tamanho de lote.

FUNÇÃO OBJETIVO

O que quero minimizar?

Custo total, erro quadrático, atraso, energia, risco ou perda operacional.

RESTRIÇÕES

O que não pode violar?

Capacidade, orçamento, prazo, limite físico, qualidade mínima ou domínio matemático.

ARGMIN

`argmin f(x)` é o valor de `x` que torna `f(x)` mínima. O mínimo é o custo; o argmin é a decisão que gera esse custo.

02GRADE

// TESTE MUITAS POSSIBILIDADES

Busca em grade é simples e confiável para começar.

Escolha um intervalo, crie candidatos e calcule a função objetivo em cada um. É fácil de entender, mas pode ser caro quando há muitas variáveis ou quando a grade precisa ser muito fina.

COMO FUNCIONA

Gera valores como `0, 10, 20...` e guarda o menor custo observado.

VANTAGEM

Não precisa de derivada e funciona mesmo quando a função tem quebras ou regras condicionais.

CUIDADO

O melhor valor pode estar entre dois pontos da grade. Refine perto dos melhores candidatos.

busca em grade
def busca_em_grade(funcao, inicio, fim, passo):
    melhor_x = inicio
    melhor_custo = funcao(inicio)
    x = inicio
    while x <= fim:
        custo = funcao(x)
        if custo < melhor_custo:
            melhor_x, melhor_custo = x, custo
        x += passo
    return melhor_x, melhor_custo
03TERNÁRIA

// QUANDO A CURVA TEM UM VALE

Busca ternária reduz o intervalo pelo formato da função.

Se a função é unimodal no intervalo, isto é, desce até um ponto e depois sobe, compare dois pontos internos e descarte o lado pior. É útil para uma variável contínua sem derivada.

ESQUERDAl
MEIO 1m1
MEIO 2m2
DIREITAr
HIPÓTESE

A função deve ter um único vale relevante no intervalo analisado.

USO

Boa escolha para ajustar um número quando o custo é suave, mas a derivada não está disponível.

PARADA

Pare quando o intervalo ficar pequeno o suficiente para a precisão desejada.

04GRADIENTE

// MOVA-SE NA DIREÇÃO QUE REDUZ CUSTO

Descida do gradiente usa a inclinação.

A derivada indica como o custo muda quando `x` muda um pouco. Para minimizar, andamos contra essa inclinação: `x_novo = x_atual - taxa × gradiente`.

GRADIENTE POSITIVO

Custo sobe se x aumenta

Diminua `x` para tentar reduzir o custo.

GRADIENTE NEGATIVO

Custo cai se x aumenta

Aumente `x` para tentar reduzir o custo.

TAXA DE APRENDIZADO

Tamanho do passo

Grande demais oscila; pequena demais demora.

NÃO É MÁGICA

Gradiente funciona melhor quando a função é suave e a taxa é bem escolhida. Em funções com muitos vales, ele pode parar em um mínimo local.

05COMPARAR

// MÉTODO DEPENDE DA FUNÇÃO

Escolha o método pelo custo da busca e pelas hipóteses.

Use busca em grade como baseline, busca ternária quando há uma variável e um vale claro, e gradiente quando você tem derivada ou aproximação confiável.

MÉTODOBusca em grade

Robusta, simples e lenta quando o espaço cresce.

MÉTODOBusca ternária

Eficiente para função unimodal de uma variável.

MÉTODODescida do gradiente

Escala melhor, mas depende de taxa, derivada e superfície.

otimizacao_custos.py
def custo(q):
    return 900 + 17.5 * q + 0.08 * (q - 140) ** 2

def gradiente(q):
    return 17.5 + 0.16 * (q - 140)

q = 20
for _ in range(20):
    q = q - 0.08 * gradiente(q)
