Dijkstra
sem chute.

Dijkstra resolve menor caminho quando os custos das arestas são não negativos. A ideia central é simples e poderosa: sempre finalize o nó com menor distância acumulada conhecida.

01MODELO

// GRAFO PONDERADO

Transforme o problema em nós, arestas e pesos.

Dijkstra trabalha sobre um grafo `G = (V, E)` com uma função de peso `w(u,v) >= 0`. Um caminho `P = (v0, v1, ..., vk)` tem custo igual à soma dos pesos percorridos.

FUNÇÃO OBJETIVO
dist(s,t) = minP:s→t Σ w(vi, vi+1)

O algoritmo não procura o caminho com menos arestas. Ele procura a menor soma de custos. Se uma rota tem três trechos baratos, ela pode vencer uma rota com um único trecho caro.

NÓ

Estado discreto

Cidade, etapa de processo, página, célula de mapa ou qualquer posição alcançável.

ARESTA

Transição permitida

Conexão entre estados. Pode representar deslocamento, dependência ou troca de estado.

PESO

Custo local

Distância, tempo, risco, energia, dinheiro ou penalidade operacional.

RESTRIÇÃO

Sem pesos negativos

Com custo negativo, um caminho ainda não finalizado pode ficar melhor depois e quebra a garantia.

02RELAXAR

// ATUALIZE O MELHOR CUSTO CONHECIDO

A operação fundamental é relaxar uma aresta.

Para cada vizinho `v` do nó atual `u`, teste se passar por `u` melhora o custo conhecido de `v`. A fila de prioridade guarda os candidatos ordenados por menor distância acumulada.

REGRA DE RELAXAMENTO
d[v] ← min(d[v], d[u] + w(u,v))

Quando `u` sai da fila com a menor prioridade, `d[u]` vira definitivo. Essa afirmação depende de `w(u,v) >= 0`: nenhum caminho futuro consegue voltar e diminuir `d[u]` sem antes passar por um custo não negativo.

03EXEMPLO

// PASSO A PASSO

Observe a fronteira encolher até o destino.

No grafo padrão do laboratório, a origem é `A` e o destino é `G`. A cada iteração, o algoritmo retira da fila o nó com menor `d` e relaxa seus vizinhos.

PassoNó finalizadoAtualizações relevantesInterpretação
1A, d=0d[B]=2, d[C]=5B entra na frente porque custa menos que C.
2B, d=2d[D]=4, d[E]=8O melhor caminho parcial para D passa por B.

INVARIANTE

Antes de cada extração, todo nó já finalizado tem distância ótima. A prova usa contradição: se existisse caminho melhor para `u`, ele teria que passar por algum nó ainda não finalizado com distância menor, que teria saído da fila antes.

04CUSTO

// QUANTO O ALGORITMO CUSTA?

A estrutura da fila muda a complexidade.

O resultado é exato, mas o custo computacional depende da implementação e da densidade do grafo.

LISTA + BUSCA LINEAR

O(|V|²)

Simples de implementar. Pode ser aceitável em grafos pequenos ou muito densos.

HEAP BINÁRIO

O((|V| + |E|) log |V|)

Boa escolha prática para grafos esparsos, como mapas e redes.

MEMÓRIA

O(|V| + |E|)

Armazena grafo, distâncias, predecessores, visitados e itens da fronteira.

QUANDO USAR

Use Dijkstra quando você precisa de ótimo garantido, tem pesos não negativos e não possui uma heurística confiável para guiar a busca até o destino.

05CÓDIGO

// IMPLEMENTAÇÃO ESSENCIAL

O código segue diretamente a matemática.

`distancias` guarda `d[v]`, `anterior` reconstrói o caminho e a heap escolhe sempre o menor custo acumulado.

dijkstra.py
def dijkstra(grafo, origem, destino):
    fila = [(0, origem)]
    distancias = {origem: 0}
    anterior = {}

    while fila:
        custo, no = heappop(fila)
        if no == destino:
            return reconstruir(anterior, origem, destino), custo

        for vizinho, peso in grafo[no]:
            novo = custo + peso
            if novo < distancias.get(vizinho, float("inf")):
                distancias[vizinho] = novo
                anterior[vizinho] = no
                heappush(fila, (novo, vizinho))
