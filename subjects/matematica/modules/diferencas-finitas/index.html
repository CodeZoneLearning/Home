FINITE_DIFFERENCES · AX_EQUALS_B

Transforme derivadas
em sistemas.

Diferenças finitas aproximam derivadas usando pontos vizinhos. O resultado prático é sempre uma equação linear: `A x = b`. Se houver apenas uma incógnita, `x = b / A`.

GRID_TO_SYSTEM
A x = b

vizinhos conhecidos entram em `b`; incógnitas entram em `x`.

01 · Malha02 · 1D03 · Sistema04 · 2D05 · Código
01MALHA

// DERIVADA VIRA DIFERENÇA

A ideia é trocar uma curva contínua por pontos.

Quando uma equação usa derivadas, como temperatura, pressão, deslocamento ou concentração, podemos calcular valores apenas em pontos de uma malha. A derivada deixa de ser simbólica e vira uma relação entre vizinhos.

PASSO H

Distância entre pontos

Quanto menor `h`, mais fina a malha. A aproximação costuma melhorar, mas o sistema fica maior.

INCÓGNITA

O que queremos descobrir

Cada ponto interno desconhecido vira uma posição no vetor `x`.

CONTORNO

O que já sabemos

Valores nas bordas são conhecidos e entram no lado direito `b`.

REGRA CENTRAL

Para a segunda derivada em 1D: `u''(x_i) ≈ (u_{i-1} - 2u_i + u_{i+1}) / h²`. Reorganizando, os coeficientes viram a linha da matriz `A`.

021D

// UM PONTO INTERNO

No caso mais simples, a conta cai direto em `x = b / A`.

Imagine uma barra com as extremidades conhecidas e apenas o ponto do meio desconhecido. Para a equação `-u'' = f`, a aproximação fica `2u_1 - u_0 - u_2 = h² f_1`. Como `u_0` e `u_2` são conhecidos, eles vão para `b`.

03SISTEMA

// VÁRIOS PONTOS INTERNOS

Com mais incógnitas, `A` deixa de ser número e vira matriz.

A lógica continua a mesma: coeficientes dos pontos desconhecidos ficam em `A`, valores conhecidos e fontes ficam em `b`, e os valores procurados ficam em `x`. Não fazemos divisão direta por matriz; resolvemos o sistema linear.

DIAGONAL

O próprio ponto

O coeficiente central costuma ser maior, como `2` em 1D ou `4` em 2D.

VIZINHOS

Acoplamento local

Pontos vizinhos aparecem com coeficientes negativos, como `-1`.

SOLVER

Resolver `A x = b`

Use eliminação, fatoração, método iterativo ou uma rotina numérica confiável.

042D

// UMA PLACA EM VEZ DE UMA BARRA

Em 2D, cada ponto conversa com quatro vizinhos.

Para uma malha plana, a aproximação clássica de `-Δu = f` usa esquerda, direita, baixo e cima. Em um único ponto interno, a equação fica `4u_centro = h²f + u_esq + u_dir + u_baixo + u_cima`.

MESMA FORMA

Com muitos pontos internos em 2D, empilhamos as incógnitas em um vetor. A matriz `A` cresce, mas a regra continua sendo montar e resolver `A x = b`.

05CÓDIGO

// MONTE ANTES DE RESOLVER

O código separa três etapas: malha, matriz e solução.

Primeiro defina o passo `h` e os pontos internos. Depois monte `A` e `b`. Só então resolva o sistema para encontrar `x`, que contém os valores aproximados da solução nos pontos internos.

Escolha quantidade de pontos e condições de contorno.

02Montar `A` e `b`

03Resolver

Calcule `x` e interprete os valores no domínio original.

diferencas_finitas.py
def resolver_escalar(A, b):
    return b / A

def ponto_central_2d(oeste, leste, sul, norte, fonte, h=1):
    A = 4 / h**2
    b = fonte + (oeste + leste + sul + norte) / h**2
    return resolver_escalar(A, b)

CHECKPOINT_06

Você sabe transformar uma equação diferencial simples em sistema linear.

A rotina mental é fixa: discretizar a malha, montar `A x = b` e resolver `x`. No caso escalar, isso vira `x = b / A`.
