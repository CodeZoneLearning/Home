01ANATOMIA

// MODELO NÃO É O ALGORITMO INTEIRO

Uma função ajustada a partir de exemplos.

O algoritmo de treinamento procura parâmetros para uma representação. O artefato resultante é o modelo: uma função que recebe novas features e produz uma previsão.

DATAX, y

Exemplos disponíveis para aprender e avaliar.

→
TRAININGalgoritmo

Procedimento que ajusta os parâmetros.

→
MODELf(X; θ)

Representação ajustada e pronta para inferência.

→
PREDICTIONŷ

Resposta produzida para um novo exemplo.

θ
PARÂMETROS

Aprendidos durante o treino

Coeficientes de uma regressão, cortes de uma árvore ou pesos de uma rede neural.

model.fit(X_train, y_train)
λ
HIPERPARÂMETROS

Definidos antes ou durante a busca

Profundidade, regularização, número de vizinhos, árvores ou camadas.

max_depth=4
SEPARAÇÃO IMPORTANTE

Treinamento ajusta parâmetros. Seleção de modelo compara representações e hiperparâmetros usando validação. O conjunto de teste fica reservado para a estimativa final.

02FAMÍLIAS

// REPRESENTAÇÕES DIFERENTES

Cada família enxerga relações diferentes.

Não existe modelo melhor fora de contexto. Cada família combina capacidade, hipóteses, custo computacional e facilidade de explicação.

LINEAR

Modelos lineares

Combinam features por pesos. São baselines fortes, rápidos e explicáveis quando a relação é aproximadamente linear.

Exemplos
Linear / Logistic Regression
Cuidado
interações e não linearidade
DISTANCE

Vizinhança

Prediz usando exemplos próximos. A escala e a definição de distância passam a fazer parte do modelo.

Exemplo
K-Nearest Neighbors
Cuidado
escala e alta dimensão
TREE

Árvores

Particionam o espaço com regras. Capturam não linearidade e interações sem exigir padronização.

Exemplo
Decision Tree
Cuidado
instabilidade e overfitting
ENSEMBLE

Ensembles

Combinam vários modelos para reduzir variância ou corrigir erros sequencialmente.

Exemplos
Random Forest / Boosting
Cuidado
complexidade e latência
KERNEL

Margem e kernels

Procuram fronteiras com boa separação e podem representar relações não lineares por kernels.

Exemplo
Support Vector Machine
Cuidado
escala e custo em bases grandes
NEURAL

Redes neurais

Empilham transformações para aprender representações complexas em grandes volumes de dados.

Exemplos
MLP / CNN / Transformer
Cuidado
dados, custo e opacidade
03CAPACIDADE

// VIÉS E VARIÂNCIA

Complexidade suficiente, não máxima.

Pouca capacidade impede o modelo de capturar o sinal. Capacidade excessiva permite memorizar ruído. O ponto útil aparece quando o desempenho fora do treino é melhor.

UNDERFITTINGalto viés

Treino e validação ruins. A representação é simples demais.

→
GENERALIZAÇÃOequilíbrio

O modelo captura sinal e mantém desempenho fora do treino.

→
OVERFITTINGalta variância

Treino excelente, validação pior e gap crescente.

05ESCOLHER

// RESTRIÇÕES ANTES DO RANKING

O melhor modelo cabe no sistema.

Uma diferença pequena na métrica pode não compensar mais latência, dependências, dificuldade de explicação ou custo de manutenção.

01

Comece por um baseline

Use uma regra simples ou modelo linear para medir o ganho real de complexidade.

02

Liste hipóteses

Escala, linearidade, independência, distância e estabilidade não são detalhes.

03

Valide fora do treino

Use splits coerentes com tempo, grupos e momento da decisão.

04

Inclua operação

Latência, memória, explicabilidade, frequência de retreino e fallback.

05

Analise erros

Compare segmentos e casos concretos, não apenas a média global.

06

Prefira a menor complexidade suficiente

Complexidade precisa comprar ganho mensurável para o produto.
