Features entram.
A pergunta define a saída.

Todo treinamento começa com exemplos representados por variáveis de entrada, as features. O que separa os paradigmas é a existência de um target conhecido durante o aprendizado.

X
FEATURES · ENTRADAS

O que sabemos sobre cada exemplo?

Temperatura, vibração, pressão, tempo de operação e qualquer variável disponível no momento da decisão.

y
TARGET · RESPOSTA

O que queremos que o modelo aprenda?

Uma classe, um valor contínuo ou outra resposta observada que possa orientar os ajustes do modelo.

REGRA MENTAL

Se existe uma resposta histórica confiável que você deseja prever, comece investigando aprendizado supervisionado. Se o objetivo é descobrir organização sem respostas prontas, investigue aprendizado não supervisionado.

02COM TARGET

// APRENDIZADO SUPERVISIONADO

O erro orienta
o ajuste.

Durante o treino, o modelo produz uma previsão, compara com a resposta real e usa o erro para ajustar seus parâmetros. O objetivo não é memorizar o treino: é manter qualidade em exemplos que nunca foram vistos.

03SEM TARGET

// APRENDIZADO NÃO SUPERVISIONADO

Sem gabarito,
mas com objetivo.

Não ter target não significa analisar sem critério. O algoritmo procura uma estrutura definida pelo método e pelas features escolhidas: similaridade, densidade, componentes ou pontos fora do padrão.

01

Clusterização

Agrupa exemplos semelhantes segundo uma representação e uma medida de distância.

02

Redução de dimensão

Comprime variáveis preservando parte relevante da estrutura ou variância.

03

Detecção de anomalias

Identifica observações raras ou distantes do comportamento esperado.

INTERACTIVE_LAB

Os mesmos pontos, duas informações.

No modo supervisionado, as cores representam classes fornecidas no target. No modo não supervisionado, o algoritmo recebe apenas posições e propõe grupos por similaridade.
